This project demonstrates the development of a structured Information Security Policy aligned with the principles of an Information Security Management System (ISMS) under International Organization for Standardization ISO/IEC 27001.

The objective of this project is to simulate how organizations create and implement formal information security policies to protect sensitive information, maintain operational resilience and support regulatory compliance.

This policy was designed for a simulated fintech organization (NovaPay Fintech Ltd) to demonstrate practical Governance, Risk and Compliance (GRC) documentation skills.


Project Objectives

The goal of this project is to demonstrate how organizations:

• Establish formal information security governance

• Define organizational security objectives

• Assign roles and responsibilities for information security

• Implement core security principles

• Enforce compliance with security policies

• Support the operation of an Information Security Management System (ISMS)


Policy Structure

The Information Security Policy developed in this project includes the following sections:

1. Purpose

Explains why the information security policy exists and its role in protecting organizational information assets.

2. Scope

Defines the employees, systems, processes, and information assets covered by the policy.

3. Information Security Objectives

Establishes the security goals that guide the organization's Information Security Management System.

4. Roles and Responsibilities

Defines accountability for information security across executive management, IT teams, employees, and third-party vendors.

5. Security Principles

Establishes the core security principles of:

• Confidentiality

• Integrity

• Availability

These principles guide how information assets are protected.

6. Policy Statements

Defines key security requirements including:

• Access Control

• Data Protection

• Incident Management

• System Security

• Security Awareness

• Third-Party Security

7. Compliance

Defines expectations for adherence to the policy and consequences of non-compliance.

8. Review and Maintenance

Establishes the process for periodic policy review and continuous improvement.
